# thehatleague.com — TanStack Start build

Sister implementation of `thehatleague.com` rendered by **TanStack
Start + Vite + Nitro** instead of Next.js 16. Same brand, same
backend, same Supabase project, same Discord bot, same Twitch channel.

## Layout

```
apps/tanstack/
├── package.json
├── tsconfig.json
├── vite.config.ts
├── README.md
└── src/
    ├── router.tsx          # TanStack Router + QueryClient bootstrap
    ├── routeTree.gen.ts    # auto-generated by the router plugin
    ├── routes/             # file-based routes
    ├── components/         # SiteHeader, SiteFooter, Hero, etc.
    ├── server/             # createServerFn RPCs
    ├── lib/                # env, cookie, ranks, tracker, supabase
    └── styles/app.css      # Tailwind v4 + brand tokens
```

## Dev

From the repo root:

```bash
pnpm install
pnpm --filter thehatleague-tanstack dev   # Vite at http://localhost:3001
```

Both apps share `.env.local` at the repo root.

## Verifying

```bash
pnpm --filter thehatleague-tanstack typecheck
pnpm --filter thehatleague-tanstack build
```

## Production build

```bash
pnpm --filter thehatleague-tanstack build
PORT=3001 pnpm --filter thehatleague-tanstack start
```

Build uses the `nitro/vite` plugin with the default node-server preset
(emits `.output/server/index.mjs` + `.output/public/`). `start` reads
`.env.local` from the repo root via Node's `--env-file` flag. Deploy
anywhere that runs Node; swap the nitro preset for Cloudflare/Bun/etc.

## Supabase OAuth

Discord OAuth callback runs at `http://localhost:3001/auth/callback`.
Add that to the Supabase project's **Authentication → URL
Configuration → Redirect URLs** allowlist, otherwise sign-in throws
`Invalid redirect URL`.

## Known quirks

- **`Invalid server function ID` after restarting dev** — Vite caches
  fn IDs across restarts. Hard reload clears it.
- **Server compression** — Nitro's default Node preset doesn't gzip
  responses. Production deploys should sit behind nginx / Cloudflare /
  similar.
